①【永久延焼の矛盾】
詠唱

焔よ、巡れ——
果てなき円環を描き、虚空に広がれ。
万象を灰燼に帰すまで、燃え続けよ。

ただし、形を崩すことなかれ。
燃料を屠り、限りなく——されど、滅びることなく。

構造的矛盾

燃焼 = エネルギー散逸（エントロピー増大）

形状維持 = 状態固定

無限燃焼 + 滅びることなかれ = エネルギー保存則無視

→ 熱力学的構文衝突

②【熱力学無視の氷結】
詠唱

氷よ、凝れ——
万物を白銀に封じ、時の流れを止めよ。

ただし、熱を奪うことなかれ。
体温を保ち、命の火を絶やさず——
静寂の檻となれ。

構造的矛盾

凍結 = 吸熱過程

熱移動禁止 = 相転移不能

体温維持 + 全凍結 = 生体保存矛盾

→ 熱力学第一法則違反

→ 衝撃波例外処理欠落

Ⅱ. M-GSL の基本原則（絶対ルール）
① 因果律非破壊原則

原因なく結果を出してはならない。

熱なしに凍結不可

光源なしに照射不可

時間停止下で変化不可

② エネルギー保存則

出力には必ず対価が必要。

無限燃焼は禁止

無限再生は禁止

永久機関型構文は即 abort

③ 定義整合性原則

命令語は自己否定してはならない。

例：

「燃え続けよ」＋「滅びるな」＝矛盾

「光を出せ」＋「影を生むな」＝概念破壊

④ 実行可能性制約

構文は物理世界で実行可能でなければならない。

「詩的正しさ」は評価対象外。

Ⅲ. M-GSL 詠唱の構造形式

すべての詠唱は以下の三層で構成される。

🧩 第一層：発動命令（Invoke）

氷よ、凝れ
光よ、満ちよ

→ 対象属性の定義

🧩 第二層：出力指定（Output）

万物を灰燼に帰せ
闇を払え
世界を切り裂け

→ 効果内容の定義

🧩 第三層：制約条件（Constraint）

ただし、〜なかれ
〜のまま
〜を許すな

→ 安全制御・範囲制御

Ⅳ. 矛盾が発生するのはどこか？

すべての矛盾は 第三層（Constraint） に集中する。

つまり、

問題は「力」ではない
問題は「制約設計」
